# 見積書 Creator

## 使う場面

ユーザーが「見積書を作成して」「見積もり作って」「◯◯様の見積書お願い」など、LP/HP案件の見積書生成を指示したときに使う。テンプレート自体の編集依頼（デザイン変更など）はこのスキルの対象外。

## テンプレート（正本、直接編集しない）

- `references/quote-templates/lp-quote-template.html` — LP（1枚絵ラスターLP／スワイプ型LP）向け
- `references/quote-templates/hp-quote-template.html` — HP（Next.js実装のコーポレートサイト）向け

どちらも単体のHTMLファイルとして完結しており、ブラウザで開くと本文を直接クリックして編集でき、右上のツールバーから印刷／PDF化・編集内容の保存ができる。

## 手順

1. **LP/HPどちらの見積書か判断する。** 指示文言（「LP」「HP」）や会話の文脈（`projects/`・`portfolio/` の対象案件がNext.js実装かラスターLPか）から分かればそれに従う。分からなければ質問する。
2. **出力先にコピーする。** `exports/quotes/` 配下に `<クライアント名スラッグ>_<lp|hp>-quote_<YYYY-MM-DD>.html` として対象テンプレートをコピーする（`exports/quotes/` は既存フォルダ）。
3. **分かっている情報だけ埋める。**
   - `【クライアント名】` → 指示から分かるクライアント名・会社名
   - `【提案日 例: ...】` → 今日の日付
   - `【屋号・会社名】`「担当者名」 → 自社情報がまだ未確定の場合はプレースホルダーのまま残す（[[quote_creator_identity]] 参照。確定していれば埋める）
   - 会話で課題・目的・見積り項目や金額など具体的な情報が既に分かっていれば該当箇所も埋める。分からない部分はテンプレートの例文・目安値のまま残してよい（あとでブラウザ上で編集する前提）。
4. **報告する。** 生成したファイルパスを伝え、ブラウザで開けば本文をクリックして直接編集でき、编集後は右上「💾 編集内容を保存」でファイルに書き出せる、印刷から「🖨 印刷/PDF化」でPDF化できる、と案内する。

## 注意

- テンプレート本体 (`references/quote-templates/*.html`) は編集しない。必ず `exports/quotes/` へコピーしてから中身を埋める。
- 金額は目安値のまま残っている場合があるため、確定金額でないときはその旨をひとこと添える。
- 生成したファイルはクライアント名・金額など案件固有の情報を含むため、コミットするかどうかはユーザーに確認する（[[feedback_personal_vs_team_repo_scope]] 参照）。

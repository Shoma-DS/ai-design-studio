# LP レイアウトテンプレート集

健康食品・美容化粧品などの単品通販（D2C）LPで使われる、定番のセクション構成4パターン。
出典: [legit.co.jp 無料LP構成・テンプレート配布](https://www.legit.co.jp/landingpage_template/16590/)（株式会社レジット）

## 使う場面

- 新規LPの構成案（ワイヤーフレーム）を作る最初の段階で、ゼロから考えず定石の型から始めたいとき。
- `skills/design/lp-creator/SKILL.md` の「手順2: コピーと構成を作る」で、セクション順序の叩き台として使う。
- 単品リピート通販・サブスク系オファー（初回限定価格、定期コース）のLPに特に適している。実在サイトの模写にならないよう、構成（型）だけを参考にし、コピー・画像は案件オリジナルにする。

## 構成01: キャッチコピー先行・即オファー型

ファーストビューで惹きつけて、そのままオファー（価格訴求）まで一気に見せる短尺型。

1. サブキャッチ＋キャッチコピー（大見出し）＋モデル写真
2. ターゲット層・利用シーンを想起させる説明文
3. 商品名＋強み訴求3点（丸アイコン×3）＋商品画像
4. 期間限定キャンペーン訴求（「一番人気／◯◯定期コース」）
5. 商品詳細（商品名・容量・原産国・商品画像・通常価格）
6. 初回限定オファー帯＋特別価格の大きな数字＋送料無料訴求
7. 定期コースの大特典リスト（1, 2, 3, 4…）

向いている商材: 説明よりもビジュアル・価格インパクトで惹きたい商品、リピート通販の入口LP。

## 構成02: 口コミ・お悩み共感型

ベネフィットを見せたあと、口コミで信頼を積み上げ、悩みへの共感で引き込む型。

1. ベネフィットキャッチ（見出し）
2. 使用シーン・実績を示す写真3点並び
3. 「口コミも好評！リピートする人続出中」＋口コミ一覧（アイコン＋テキスト×複数）
4. 「こんなお悩み、ありませんか？」暗転帯＋悩みリスト（箇条書き）
5. 悩みが引き起こす悪循環の説明（問題の深掘り）

向いている商材: 信頼性・実績訴求が効く商材、悩み訴求からの導入が自然なジャンル（健康食品、美容など）。後続で構成01・03のオファーパートに接続する運用を想定。

## 構成03: ベネフィット訴求＋オファー型（構成01の亜種）

構成01と似た型だが、悩み解消の言い切りコピーと写真を先に見せてからオファーに入る。

1. 「商品名で◯◯のような」＋ベネフィットキャッチ（見出し）
2. 「もう悩まない！」言い切りコピー＋女性写真
3. 期間限定キャンペーン訴求（「一番人気／◯◯定期コース」）
4. 商品詳細（商品名・容量・原産国・商品画像・通常価格）
5. 初回限定オファー帯＋特別価格＋送料無料訴求
6. 定期コースの大特典リスト（1〜4）＋特典訴求

向いている商材: 構成01よりも「効果の言い切り」で共感を作りたい商材（ダイエット、スキンケアなど）。

## 構成04: 多機能・ベネフィット列挙型

単一の強い訴求ではなく、複数の性能・メリットを並べて納得感を積み上げる型。

1. 「しかも」＋サブメリット付き見出し（「◯◯でも使える」）
2. 商品画像（中央）＋左右に用途アイコン＋一言ベネフィット
3. 商品の性能訴求ブロック（見出し＋説明文）を複数回繰り返し
4. 「メリット」まとめ（見出し＋画像枠＋説明文）

向いている商材: 機能・成分が豊富でスペック訴求が効く商材（クレンジング、多機能コスメ、ガジェットなど）。

## 構成05: スワイプ型カルーセルLP（予約・来店誘導特化）

構成01〜04とは設計思想が異なる別ジャンル。縦スクロールではなく、Instagramストーリーズのように「1枚1画面のフルブリード画像＋テキストオーバーレイ」を矢印またはスワイプ操作でめくっていく形式。
出典: [tryhatch.co.jp スワイプLP事例9業種18選](https://lab.tryhatch.co.jp/swipe-lp/18case-study/)（GMOトライハッチ株式会社、自社ツール「SwipeKit」の事例記事）。元記事の実例は左右送りだったが、`portfolio/swipe-lp-salon-demo/`での検証時にユーザーが縦送り（TikTok/Instagramリール型の上下スワイプ）を指定し、そちらで実装・検証済み。次回作る際も特に指定がなければ縦送りをデフォルトにする。

カード構成（1カード＝1画面）:

1. フックカード: 世界観訴求コピー（「あなた史上、いちばん似合う髪へ」等）＋雰囲気を伝えるフルブリード写真／動画
2. 悩み・共感カード: ターゲットの悩みに寄り添うコピー＋実例写真
3. 実績・信頼カード: 口コミ・Before/After・実績数字
4. オファーカード: 割引・キャンペーン訴求（「初回限定 最大30%OFF」等）を大きな数字で
5. 最終CTAカード: 予約導線（LINE予約・電話・フォームへの固定CTAボタン）

共通UI要素:
- 上下（または左右）矢印ナビ（クリック）＋タッチスワイプ／マウスドラッグの両対応
- 実装はPointer Events + `setPointerCapture`で統一する（mousedown/touchstartを別々に扱うと、ポインタがカード外に出た際にリリースを取りこぼしてカルーセルが操作不能な状態で固まることがある）
- 矢印ボタンはカルーセル要素の内側に置く場合、ボタン上から始まったpointerdownではドラッグ検知・`setPointerCapture`を発火させないようにする（発火させるとボタンの`click`イベントが握りつぶされ、ボタンが反応しなくなる）
- カード下部にラベル表示（「サロン紹介①」等）でどのカードか把握できるようにする
- 画面内に小さいタグボタン（「店内映像へ」等）でサブコンテンツに寄り道させる導線
- 各カードにCTAを軽く仕込み、最後のカードで強いCTAに収束させる

型のバリエーション: 同記事では業種ごとに「テンプレートA（紹介型・世界観で見せる）」「テンプレートB（悩み・実績訴求型）」の2パターンが提示されている。構成02（悩み訴求）の考え方をカード形式に翻訳したものと捉えてよい。上記の5枚構成は「感情訴求型」（テンプレートAに近い、写真主体・コピー少なめ）。もう一つ、`https://lpcats.com/`のスワイプLP事例（士業・BtoB向け）を分析して確立した「情報訴求型」もある。

**情報訴求型**（7〜8枚。士業・コンサル・BtoBなど検討期間が長く納得感の積み上げが必要な業種向け）:

1. フックカード: 世界観訴求コピー＋ブランド名
2. 実績・数字訴求カード: 大きな数字（「〇〇%削減」等）＋実績リスト
3. 悩み・共感カード: チェックリスト形式（「1つでも当てはまったら要チェック！」）
4. 解決策カード: サービス内容を番号リスト（01, 02, 03…）で提示
5〜6. お客様の声カード: 1人1カードで職種・年代を変えて複数提示し、幅広い層に「自分ごと化」させる
7. 代表挨拶カード: 権威性（実績・資格）と親近感（顔出し・会話体の文章）を両立
8. 最終CTAカード: 期間限定オファー＋強いCTA

実績を先出しし、悩みは3枚目まで後ろにずらす点が構成02と異なる（先に信頼の土台を作ってから悩みを突く方が押し売り感が薄れる）。

向いている業種: 予約・来店が成果指標になる業種（美容室・サロン・整体・歯科・学習塾・ホテル・採用など＝感情訴求型）、士業・コンサル・BtoBなど検討期間が長い業種（＝情報訴求型）。単品通販・D2Cには不向き（構成01〜04を使う）。

実装上の注意: 構成01〜04と違い、静的なワイヤーフレーム画像だけでは検証できない。スワイプ/矢印カルーセルのJS実装（vanilla JSまたはSwiper.js等）が必要で、`lp-creator`の通常フローに乗せる前に別途カルーセルの動作確認が要る。原稿作成から画像生成・実装・公開・ギャラリー登録までの一気通貫フローは `skills/design/swipe-lp-creator/SKILL.md` を使う。

## 使い方

1. 案件のオファー内容（単発訴求か／悩み訴求か／多機能訴求か／予約・来店誘導か）に応じて上記5つから近い型を選ぶ。予約・来店系の業種なら構成05を検討する。
2. 選んだ型のセクション順を土台に、`skills/design/lp-creator/SKILL.md` の手順でコピー・画像プロンプトを作る。構成05を選んだ場合はカルーセルUIの実装検証を別途行う。
3. 必要なら複数の型のセクションを組み合わせてよい（例: 構成02の悩み訴求→構成01のオファーパート）。

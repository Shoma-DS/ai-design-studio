# LP Creator

## 使う場面

ユーザーが「LP」を作りたいと明示したときに使う。

## 手順

1. 目的、媒体、サイズ、ターゲット、トーン、必須コピーを確認する。
2. 明示されていない重要条件だけ質問する。
3. 保存先の案件フォルダを最初に作る（下記「保存先」参照）。以降の全ファイルはリポジトリ直下ではなく、このフォルダの中に作成する。
4. コピーと構成を作る。
5. `skills/design/image-prompt-generator/SKILL.md` に従って画像プロンプトを作る。
6. Codex app-server / gpt-image-2 でラスター画像を生成する。
7. 生成物を手順3で作ったフォルダ内に保存する（ルート直下に置かない）。
8. `skills/design/lp-responsive/SKILL.md` に従い、PC版に加えてスマホ版のセクション画像と`<picture>`切り替えを用意する。「レスポンシブにして」と明示されていなくても、LP制作の標準工程として必ず行う。

## 保存先

作業を始める前に、次のどちらかで案件フォルダを決めてユーザーに確認する。

- 実案件・クライアントワーク: `projects/<client-or-business>/<project-name>/`
- 実在ブランドの模写・参考由来・自主制作: `portfolio/<project-name>/`（商標・実在人物・元コピーは再現しない架空案件として保存）

フォルダ構成は既存プロジェクト（例: `portfolio/beauty-salon-c3-inspired-lp/`）に揃える。

```text
<project-name>/
  lp/index.html      ← リポジトリ直下の lp.html ではなくここに置く
  lp/images/         ← リポジトリ直下の lp_images/ ではなくここに置く
  copy/              ← LP原稿・画像プロンプトのmd
  references/         ← 参考LPのスクショなど
  outputs/           ← 完成版スクショなど
```

途中経過の切り出し画像・作業用クロップなどは `scratch/` にまとめ、`exports/` や `portfolio/` の完成品と混在させない。

## 参考スクリーンショットの保存

参考LPのスクリーンショットを撮った時点で、リポジトリ直下や `Desktop` などに置かず、その場で対象案件フォルダの `references/` へ直接保存する。

- 案件フォルダがまだ無い場合は、スクリーンショットを撮る前に「保存先」の手順で先に作る。
- 1案件で複数枚撮った場合は `reference-<内容>.png` のように用途がわかる名前を付け、後から見ても元LPが分かるようにする。
- 作業の区切り（コミット・push前など）で `git status` のuntrackedを確認し、リポジトリ直下に画像ファイルが残っていないかチェックする。残っていたら該当案件の `references/` へ移すか、対応案件が無ければユーザーに確認する。

## 注意

- 参考画像は構造・雰囲気・視線誘導のみ使う。
- 商標、実在人物、元コピーの再現は禁止。
- 文字切れ、可読性、余白、CTAの視認性を確認する。

## 配色トーンの決め方（重要）

画像プロンプトの【色・雰囲気ルール】は、参考画像を実際に見て、その彩度・コントラスト・塗りの質感（フラットなベタ塗りか、写真ベースか、水彩・パステルか）を言語化してから書く。「パステル〜ビビッドの中間トーン」「手描き風のあたたかいタッチ」のような曖昧なデフォルト表現を、参考画像を確認せずに流用しない。

- 参考画像が高彩度・フラットな塗り・輪郭線のはっきりしたイラスト（子ども向けキャンペーン、ポップな販促物など）なら、色・雰囲気ルールにも「参考画像と同等の高彩度・フラットな塗り・はっきりした輪郭線」を明記し、水彩・グラデーションのにじみ・くすみ表現は避ける指示を入れる。
- 参考画像が写真ベースで淡いパステル・ソフトフォーカスなもの（美容・コスメ系など）なら、その淡さは意図されたトーンなので、そのまま踏襲してよい。
- 生成後は必ず参考画像とセクション画像を並べて見比べ、彩度・コントラストの印象が近いか確認する。ずれている場合は色・雰囲気ルールを具体的に修正してから再生成する。

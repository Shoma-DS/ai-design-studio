# Vibe Coding 用語集

「それ、なんて言えばいいの？」を解消するための、Web制作パーツ・スタイル・インタラクションの日本語⇔英語対応表。
出典: [vibecoding-glossary.pages.dev](https://vibecoding-glossary.pages.dev/)（川合卓也氏制作）

## 使う場面

- ユーザーが技術用語ではなく感覚的な言葉（「上のあの帯」「ふわっと」「四角いあれ」）で指示してきたとき、意図を正確なUI/CSS用語に翻訳して確認・実装するために使う。
- コードやプロンプト（画像生成・LP構成案）に正確な部位名・プロパティ名を使いたいとき。
- `skills/design/ui-pattern-dictionary/SKILL.md`（UIパターンの型）と対で使う。こちらは「パーツ・プロパティの名前」、あちらは「UIパターンの型」を扱う。

### ページ構造

- **ヘッダー / Header** — ページ最上部に固定される帯状のエリア。ロゴ・ナビゲーション・CTAボタンなどを配置する、サイトの「顔」。
- **フッター / Footer** — ページ最下部に配置されるエリア。著作権表示、サイトマップリンク、SNSアイコン、会社情報などを置く。
- **サイドバー / Sidebar** — メインコンテンツの横に配置される縦長のエリア。カテゴリ一覧、関連記事、広告、フィルターなどを置く。
- **セクション / Section** — ページを意味のある単位で区切る大きなブロック。「ヒーローセクション」「特徴セクション」「料金セクション」など。
- **ヒーローセクション / Hero Section** — ページを開いた瞬間に目に入る、最初の大きなエリア。キャッチコピー、サブテキスト、CTAボタン、背景画像を配置する。
- **コンテナ / Container** — コンテンツの最大幅を制限し、左右に余白を確保する透明な箱。大画面で文章が横に広がりすぎるのを防ぐ。
- **CTA（シーティーエー） / Call To Action** — ユーザーに行動を促す要素。「今すぐ申し込む」「無料で始める」などのボタンやセクション全体を指す。

### ナビゲーション

- **ナビゲーションバー / Navigation Bar / Navbar** — ページ上部にある横並びのメニュー。サイト内の主要ページへのリンクを並べる。
- **ハンバーガーメニュー / Hamburger Menu** — 三本線のアイコン（☰）をタップするとメニューが開くUIパターン。主にスマホ表示で使われる。
- **パンくずリスト / Breadcrumb** — 今いるページの階層を「ホーム > カテゴリ > 記事名」のように表示するナビゲーション。
- **ページネーション / Pagination** — コンテンツを複数ページに分割し、「1 2 3 ... 次へ」と移動できるUI。一覧ページでよく使う。
- **タブ / Tab** — 複数のコンテンツを同じ場所で切り替えて表示するUI。料金プラン（月額/年額）の切り替えなどに使う。
- **アンカーリンク / Anchor Link** — 同じページ内の別の位置にジャンプするリンク。ナビから各セクションに飛ばすLPの定番。

### UIパーツ

- **ボタン / Button** — クリック/タップで何かのアクションを実行するUI要素。リンクとは異なり「操作」を想定した要素。
- **カード / Card** — 画像・タイトル・テキストをひとまとめにした箱型のUI。記事一覧、商品一覧、チーム紹介などで使う。
- **モーダル / Modal / Dialog** — 画面の上に重ねて表示される小窓。背景が暗くなり、操作はモーダル内に集中する。確認・入力・詳細表示に使う。
- **ドロップダウン / Dropdown** — クリックやホバーで下方向に展開されるメニューやリスト。ナビのサブメニュー、選択肢の表示に使う。
- **アコーディオン / Accordion** — タイトルをクリックすると中身が開閉するUI。FAQ、詳細情報の表示に最適。
- **ツールチップ / Tooltip** — 要素にマウスを乗せると表示される小さな説明テキスト。ボタンやアイコンの補足説明に使う。
- **トースト通知 / Toast / Snackbar** — 画面の隅に一時的に表示される通知メッセージ。「保存しました」「送信完了」など操作の結果を伝える。
- **バッジ / Badge** — 小さな丸や長丸で状態・数量・カテゴリを示す要素。「NEW」「3件」「完了」など。
- **アバター / Avatar** — ユーザーのプロフィール写真やイニシャルを丸く表示するUI。チャット、レビュー、メンバー一覧で使う。
- **タグ / Tag / Chip** — カテゴリやキーワードを小さなピル型で表示する要素。ブログのカテゴリ、スキルタグなどに使う。
- **プログレスバー / Progress Bar** — 進捗状況を横棒のグラフで視覚的に表示するUI。スキル表示、アンケート結果、読了率に使う。
- **スピナー / ローディング / Spinner / Loading** — データ読み込み中を示す回転アニメーション。くるくる回る円や、スケルトン表示など。
- **カルーセル / スライダー / Carousel / Slider** — 複数の画像やカードを横スクロールで切り替えるUI。ヒーロー画像、お客様の声、商品一覧で使う。
- **ディバイダー / 区切り線 / Divider / Separator** — コンテンツ間を視覚的に区切る水平線。セクション区切り、リスト項目の区切りに使う。
- **スケルトン / Skeleton Screen** — 読み込み中にコンテンツの「骨格」をグレーのブロックで表示するUI。スピナーより体感が速い。

### フォーム

- **テキスト入力欄 / Input / Text Field** — 1行のテキストを入力するフォーム要素。名前、メールアドレス、検索ワードなどの入力に使う。
- **テキストエリア / Textarea** — 複数行のテキストを入力できる大きな入力欄。お問い合わせのメッセージ欄などに使う。
- **チェックボックス / Checkbox** — 複数選択可能な四角いチェック欄。利用規約への同意、複数項目の選択などに使う。
- **ラジオボタン / Radio Button** — 選択肢の中から1つだけ選ぶ丸いボタン。性別、プラン選択など排他的な選択に使う。
- **セレクトボックス / Select / Dropdown** — クリックすると選択肢が展開されるフォーム要素。都道府県選択、カテゴリ選択などに使う。
- **トグルスイッチ / Toggle Switch** — ON/OFFを切り替えるスライド式スイッチ。ダークモード切替、通知設定などに使う。
- **ラベル / Label** — 入力欄の上や横に表示される項目名。「お名前」「メールアドレス」など、何を入力すべきかを示す。
- **プレースホルダー / Placeholder** — 入力欄に薄い文字で表示される入力例やヒント。文字を打ち始めると消える。
- **バリデーション / Validation** — 入力内容が正しいかチェックする仕組み。エラーがあれば赤文字で「メールアドレスの形式が正しくありません」などと表示する。

### インタラクション

- **ホバー / Hover** — マウスカーソルを要素の上に乗せた状態。色が変わる、浮き上がる、下線が出るなどの視覚変化を付ける。
- **クリック / タップ / Click / Tap** — マウスでクリック、またはスマホで画面をタップする操作。ボタンやリンクの基本的な操作。
- **スクロール / Scroll** — ページを上下（または横）に移動する操作。スクロールに連動したアニメーションやパララックス効果が人気。
- **トランジション / Transition** — CSSプロパティの変化をなめらかにする仕組み。色・サイズ・位置などが「パッ」ではなく「ふわっ」と変わる。
- **アニメーション / Animation** — 要素を動かす視覚効果。フェードイン、スライドイン、バウンス、回転など。トランジションより複雑な動きが可能。
- **フォーカス / Focus** — 入力欄やボタンが「今操作中」の状態。入力欄をクリックしたとき枠が青く光るあの状態。
- **ドラッグ＆ドロップ / Drag & Drop** — 要素をマウスで掴んで別の場所に移動する操作。タスクボード、画像アップロード、リスト並び替えに使う。
- **スワイプ / Swipe** — スマホで指を横にスライドする操作。カルーセル、画像ギャラリー、削除操作などで使う。

### レイアウト

- **フレックスボックス / Flexbox** — 要素を柔軟に横並び・縦並びにするCSSレイアウト。最もよく使うレイアウト手法。
- **グリッド / CSS Grid** — 行と列で要素を格子状に配置するCSSレイアウト。2次元の配置が得意で、カードの等間隔配置に最適。
- **レスポンシブ / Responsive Design** — 画面サイズ（PC・タブレット・スマホ）に応じてレイアウトを自動的に変化させるデザイン手法。
- **ブレークポイント / Breakpoint** — レスポンシブ対応でレイアウトが切り替わる画面幅の境界値。768px（タブレット）、1024px（PC）など。
- **マージン / Margin** — 要素の「外側」の余白。要素と要素の間隔を空ける。
- **パディング / Padding** — 要素の「内側」の余白。要素の枠とコンテンツ（文字や画像）の間に空白を作る。
- **ギャップ / Gap** — FlexboxやGridで、子要素同士の間隔を統一的に設定するプロパティ。marginより簡潔に書ける。
- **重なり順（z-index） / z-index** — 要素の重なり順を制御する数値。数値が大きいほど前面（手前）に表示される。モーダルやドロップダウンで必須。

### 文字・テキスト

- **フォント / Font / Font Family** — 文字の書体。「ゴシック体」（見出し・UI向き）「明朝体」（本文・高級感）「等幅フォント」（コード用）など。
- **フォントサイズ / Font Size** — 文字の大きさ。px（ピクセル）、rem（基準サイズの倍率）、clamp()（可変サイズ）などで指定。
- **太さ（ウェイト） / Font Weight** — 文字の太さ。400が標準、700がbold。100〜900まで9段階ある（フォントが対応している場合）。
- **行間（行の高さ） / Line Height** — テキスト行と行の間隔。1.5〜1.8が読みやすいとされる。1だと行がくっつき、2だと離れすぎる。
- **文字間（字送り） / Letter Spacing** — 文字と文字の間隔。見出しは少し広め、本文はデフォルトが一般的。
- **テキスト揃え / Text Align** — テキストの水平方向の揃え位置。左揃え（left）、中央揃え（center）、右揃え（right）、両端揃え（justify）。
- **見出し（h1〜h6） / Heading** — ページ内のタイトル・小見出しを示すHTML要素。h1が最も大きく重要、h6が最も小さい。SEOにも影響する。
- **テキスト省略 / Text Truncation / Ellipsis** — テキストが長い場合に「...」で省略する処理。カードの説明文、一覧表示で文字がはみ出すのを防ぐ。

### 色・装飾

- **背景色 / Background Color** — 要素の背景の色。セクションごとに背景色を変えるとメリハリが出る。
- **グラデーション / Gradient** — 色がなめらかに変化する背景。2色以上を指定して方向（上下、左右、斜め）を決める。
- **シャドウ（影） / Box Shadow / Drop Shadow** — 要素に影を付ける装飾。浮いている感じ、奥行き感を演出する。カード、ボタン、モーダルで定番。
- **ボーダー（枠線） / Border** — 要素の周りに描かれる線。太さ、スタイル（実線/破線/点線）、色を指定できる。
- **角丸（ボーダーラジウス） / Border Radius** — 要素の角を丸くする装飾。値が大きいほど丸くなる。50%で完全な円になる。
- **透明度（オパシティ） / Opacity** — 要素の透明度。1が完全に見える、0が完全に透明。0.5で半透明。背景やオーバーレイで使う。
- **すりガラス効果 / Backdrop Blur / Glassmorphism** — 背景をぼかして、すりガラスのような半透明効果を作る。ヘッダー、モーダル背景、カードに使う。

### メディア

- **画像 / Image (img)** — ページに画像を表示する要素。サイズ指定、レスポンシブ対応（max-width:100%）、alt属性が重要。
- **動画 / Video** — ページに動画を埋め込む。自前の動画ファイルか、YouTubeなどの外部サービスのiframe埋め込みが一般的。
- **アイコン / Icon** — 小さな絵柄で意味を伝えるUI要素。矢印、ハンバーガー、検索、チェックマークなど。SVGかアイコンフォントで表示。
- **ファビコン / Favicon** — ブラウザのタブに表示される小さなアイコン。ブランド認知とタブの識別に重要。
- **OGP画像 / Open Graph Image** — URLをSNSでシェアしたときに表示されるサムネイル画像。1200x630pxが推奨サイズ。
- **画像の表示方法 / Object Fit** — 指定サイズの枠内で画像をどう表示するか制御するCSS。coverで枠を埋める、containで全体を表示。

### 開発基礎

- **コンポーネント / Component** — 再利用可能なUIの部品。ボタン、カード、ヘッダーなどをコンポーネントとして作り、何度でも使い回す。
- **API（エーピーアイ） / API (Application Programming Interface)** — 外部サービスとデータをやり取りする仕組み。天気情報の取得、決済処理、SNS連携などで使う。
- **デプロイ（公開） / Deploy** — 作ったWebサイトをインターネット上に公開すること。Vercel、Netlify、GitHub Pagesなどのサービスを使う。
- **npm / パッケージ / npm (Node Package Manager)** — JavaScriptのパッケージ（便利な部品集）を管理するツール。必要な機能をインストールして使う。
- **DOM（ドム） / DOM (Document Object Model)** — HTMLをブラウザが解釈してできる要素のツリー構造。JavaScriptでDOMを操作して、ページを動的に変更する。
- **イベント / Event** — ユーザーの操作（クリック、入力、スクロールなど）を検知する仕組み。「ボタンを押したら〇〇する」の「押したら」部分。
- **CSS変数 / CSS Custom Properties / Variables** — 色やサイズなどの値に名前を付けて一括管理する仕組み。「--primary: #7c6aef」と定義して何度でも使い回せる。
- **クラス（class） / CSS Class** — HTMLの要素にスタイルの名前を付ける仕組み。同じクラス名を付ければ同じデザインが適用される。
- **ダークモード / Dark Mode** — 背景を暗く、文字を明るくする配色モード。目の疲れ軽減、バッテリー節約。トグルスイッチで切り替える。
- **アクセシビリティ / Accessibility (a11y)** — 障害の有無に関わらず誰でも使えるようにする設計指針。alt属性、キーボード操作、コントラスト比などが重要。

## 使い方の例

- ユーザー「あの、ふわっと出てくる小窓を閉じるボタンつけて」→ **モーダル(Modal)** の **トランジション(Transition)** 付き実装、**Close** ボタンとして理解する。
- ユーザー「スマホで見た時に三本線のあれにして」→ **ハンバーガーメニュー(Hamburger Menu)** としてスマホ幅の**ブレークポイント(Breakpoint)** で切り替える実装を提案する。
- ユーザー「文字と文字の間隔を少し広げて」→ **文字間（字送り / Letter Spacing）** の調整と理解する（行間 = Line Heightと混同しない）。

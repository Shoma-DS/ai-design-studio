# LP レスポンシブ対応

このスキルは、`lp-creator` で作ったLP（セクション画像を縦結合した1枚のラスターLP）を、PC幅とスマホ幅の両方できちんと読めるようにするために使う。

**新規LPを作るときは、`lp-creator` の手順の最後に必ずこのスキルを適用する。** 「レスポンシブにして」と明示的に頼まれたときだけでなく、LP制作そのものの標準工程に含める。

---

## 前提: このスタジオのLPは「1枚のラスター画像」

`portfolio/<project>/lp/index.html` は、セクション画像を縦結合した `lp-full.png` を1枚の `<img>` として表示するだけの構造。テキストは画像に焼き込まれているため、ブラウザの標準的なテキスト折り返しやフォントサイズ調整は効かない。**レスポンシブ対応＝画面幅に応じて適切なレイアウトの画像に切り替えること。**

**スマホ版が正本、PC版はスマホ版から派生させる（2026-07-11からユーザー承認済み）。** 以前は「PC版を16:9の横長に固定する」方針だったが、単純に横へ引き伸ばした不自然なレイアウトになったためユーザーが差し戻した。今の方針は次の通り。

- **PC版セクション画像は縦横比を固定しない。** 16:9のような決め打ちの横長比率は使わない。セクションの内容量に応じて自然な高さになるようにする（テキスト・リスト中心のセクションはポートレートに近いままでよく、比較・並列コンテンツが実際にあるセクションだけ横に広がってよい）。
- **PC版の構図は、スマホ版の情報設計（要素の並び順・余白のリズム・視線導線・アイコンや挿絵のスタイル）を基準にする。** スマホ版を単純拡大したのでも、まったく別のレイアウト言語に作り替えたのでもなく、「スマホ版の世界観を保ったまま少し余白の効いたPC版」を目指す。
- **PC版コンテナは `max-width: 1200px` で中央配置**（旧: `width: min(100%, 1080px)`）。1200px以上の画面では左右に余白が出る。横スクロールは発生させない。
- `server/codexImageClient.mjs` のデフォルトプロンプトは、この「スマホ版基準・1200px上限・固定比率なし」の方針を反映済み。

---

## Step 1 — 再生成が必要かどうかを判定する

全セクションを機械的に作り直す前に、まず数枚のセクション画像を実際に見て判断する（`Read`ツールで画像を開く）。

判定基準（PC版）:

- **既にスマホ版の情報設計を踏まえた自然な構図で、1200px幅で表示しても間延び・不自然な横伸びがない** → 再生成は不要。
- **16:9などの固定比率に強制されている、またはスマホ版と印象が大きく異なる横長バナー化している** → 下記「PC版をスマホ版基準で作り直す」で作り直す。

判定基準（スマホ版）:

- **既に1080px前後の縦長・単一カラムで、見出し・本文とも画面幅に対して十分大きい** → 既にスマホ向けレイアウトなので、**再生成は不要**。無駄なAI生成コストと時間をかけない。
- **左右2カラム（テキスト+画像を横並び）、横一列に3つ以上のカード/アイコン、比較表など、PC画面前提の横方向レイアウトが実際にある** → 該当セクションのみスマホ版を再生成する。

判断はセクションごとに行ってよい。

## PC版をスマホ版基準で作り直す

1. **スマホ版セクション画像を参照画像として使う**（`lp/images/mobile/<section.imageName>` があればそれ、無ければ既存のスマホ寄りの縦長画像でよい）。PC版の古い横長画像は参照にしない。
2. `buildPrompt(section)` の結果に、以下のPC版レイアウト指示を追記してプロンプトを作る。

```
────────────────
【PC版レイアウト再構成の指示（最優先・厳守）】
このセクションは【PC/デスクトップ表示専用のセクション画像、max-width 1200pxのコンテナに中央配置される】として出力する。
・添付する参照画像（同一セクションのスマホ版）の情報設計・要素の並び順・余白のリズム・視線導線・アイコンや挿絵のスタイルを踏襲する
・スマホ版を単純に横へ引き伸ばした横長バナーにはしない。縦横比は16:9などに固定せず、このセクションの内容量に合った自然な高さにする（テキスト中心ならポートレートに近いままでよい。実際に比較・並列させる内容がある場合だけ横に広げる）
・コピー・ブランド要素・配色・情報量はスマホ版と完全に同じに保つ（情報を削らない・足さない）
・1200px幅で表示したときに間延びせず、スマホ版と一貫した世界観に見える構図にする
・上下端は前後セクションへ自然につながる余白で終える
```

3. 生成した画像は必ず`Read`で開き、(a) スマホ版と印象が近いか (b) 1200px幅で不自然に間延びしていないか (c) コピーが欠落・見切れていないか (d) 次セクションの要素が写り込んでいないか を確認する。
4. 古いPC版画像は上書きせず `references/pre-<変更内容>/` などに退避してから差し替える（後で見比べられるようにする）。
5. `lp/style.css` の `.lp-shell` を `max-width: 1200px; margin: 0 auto; width: 100%;` にする（`width: min(100%, 1080px)` などの古い指定から更新する）。セクション画像自体は `width: 100%; max-width: 1200px; height: auto;` で縦横比を保ったまま表示する。

## Step 2 — スマホ版セクション画像を生成する（無ければ先にこちらを作る）

**スマホ版は正本なので、`buildPrompt(section)` から直接生成する**（PC版からの変換ではない）。既に `lp/images/mobile/<section.imageName>` があり、内容量に対して十分読みやすい縦1カラムなら再生成不要（Step 1参照）。

再生成が必要なセクションについて、`server/codexImageClient.mjs` の `generateImageWithCodexAppServer` を使う（`skills/common/codex-image-auth/SKILL.md` の認証確認手順に従う）。

`buildPrompt(section)` の結果に、以下のスマホ向け指示を追記してプロンプトを作る。既存のPC版画像がある場合は参照画像として渡してよい（配色・ブランド・コピーの一貫性を保つため）が、なければ無しで構わない。

```
────────────────
【スマートフォン専用レイアウトの指示（最優先・厳守）】
このセクションは【スマートフォン縦画面専用】のセクション画像として出力する。
・左右2カラムの要素は、必ず縦1カラムに積む（左右に並べない）
・横並びのアイコン・カード・ボタンが3つ以上ある場合は、2列グリッドまたは縦積みに構成する（横一列にしない）
・見出し・本文とも、スマートフォンで実寸表示した際に十分大きく読める文字サイズにする
・750px幅のスマートフォン縦型LPセクションとして出力し、高さは内容に合わせて自然に調整する
・上下端は前後セクションへ自然につながる余白で終える
```

出力先: `lp/images/mobile/<section.imageName>`

このスマホ版が、上記「PC版をスマホ版基準で作り直す」の参照元になる。

**生成した画像は必ずReadツールで開き、下端に次セクションの要素が見切れて写り込んでいないか確認する。** 見つかったら下端をクロップして直す（AI画像生成はセクション境界を厳密に切れないことがある）。

## Step 3 — スマホ版を1枚に結合する

PC版と同じ結合ロジックを使う。

1. 各スマホ版セクション画像を `sips -s format png --resampleWidth 1080` で幅1080pxに揃える。
2. `pngjs`（`node_modules/pngjs/lib/png.js`）で `PNG.bitblt` を使い、順番に縦結合する。
3. `lp/images/mobile/lp-full-mobile.png` として保存する。

再生成不要と判定したセクションは、PC版の同じ画像（1080px幅にリサイズ済みのもの）をそのまま結合に使ってよい。

## Step 4 — HTMLをpicture要素で切り替える

`lp/index.html` の `<img class="lp-full">` を `<picture>` で包み、767px以下でモバイル版に切り替える。

```html
<picture>
  <source media="(max-width: 767px)" srcset="./images/mobile/lp-full-mobile.png">
  <img class="lp-full" src="./images/lp-full.png" alt="...">
</picture>
```

`.lp-shell` は `max-width: 1200px; margin: 0 auto; width: 100%;` にする（1200px以上の画面で左右に余白が出るように）。`.lp-full` は `width: 100%; max-width: 1200px; height: auto;` で縦横比を保つ。

## Step 5 — 動作確認

- ローカルで `python3 -m http.server` 等を立てて開き、ブラウザ幅を768px前後で切り替えて画像が変わることを確認する（自動化ツールのウィンドウリサイズが効かない場合は、ユーザーに実ブラウザでの確認を依頼する）。
- Vercelにデプロイ済みの場合は、Preview URLで確認してから本番反映する。

## Step 6 — ギャラリーでのプレビュー確認

`gallery/` サイトのカードをクリックすると、PC/スマホ切り替えボタン付きのプレビューモーダルが開く（`gallery/script.js` の `openPreview` / `setDevice` 実装済み）。新しいLPをギャラリーに追加した際は、このプレビューでも表示が崩れていないか確認する。
